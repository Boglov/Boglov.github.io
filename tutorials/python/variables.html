Variables and Data Types

~ What are variables? ~

In case you're not familiar with the concept of variables, they are simply containers which can hold your data so it can be used later in your code. To create a variable you just have to come up with a name for it which describes what it will be storing, and set it equal to the value/data you wish to store inside it. For example: 

1
2
# variable_name = data
favorite_number = 28

~ Commenting your code ~

Before we get into variables any further I thought it would be important to introduce you to comments. The first line in the code above is what's called a comment. A comment is just a description meant to clarify what your code is doing and while it isn't necessary to comment your code, it is very wise. Comments are not specific to variables though, you can use them to describe any piece of code you'd like to touch on.

Commenting your code makes it easier, for you or anyone else who reads your code, to understand what that code is doing. In Python a comment is begun with a pound sign, #. For better readability you should follow the pound sign by one space before typing in your comment as I've done above.

Keep in mind that comments have no effect on the functionality of your code and are actually ignored by the Python interpereter.

~ Naming your variables ~

One important thing about coding is how you name your variables.The most widely used format is to use either one lowercase letter, word, or words seperated with underscores. As we have done with the variable my_phrase in the "Hello, World!" tutorial. Other examples would be x, var or my_variable.

When you are deciding what to name variables it's ideal to come up with a name that describes what data it holds. For example, in a video game, you would probably name the players health variable "health". You may also make a variable that stores how many gold pieces your player has.

In that case it's probably better to name the variable gold_pieces rather than gp for the sake of being descriptive instead of trying to crunch it all down to the shortest possible name. So try to come up with a name with name that's not too long, but don't worry about all your variable names being super short.

~ Basic data types ~

In the last tutorial we learned how to store a line of text in a variable we named my_phrase. Text is not the only thing you can store in variables though, variables can hold all sorts of data which can be used later when it is needed.

Some data types which variables can hold include integers, floats, strings, and booleans. It's very easy to create variables to hold your data, and it's a key part of programming. Below are a few examples:

# Storing an integer (whole number)

my_float = 3.75

~ More complex data types ~

~ Tuples ~

Most other data types are just combinations of these basic ones shown above, and are technically called data structures. An example of a data structure is what's called a tuple. Tuples consist of multiple pieces of data stored together seperated by commas. Like so: 

 1
 2
 3
 4
 5
 6
 7
 8

# Storing a tuple of integers (multiple integers seperated by commas)
my_integer_tuple = (3, 5, 8, 2)
print("Integer tuple:", my_integer_tuple)

# Storing a tuple of floats (multiple floats seperated by commas)
my_float_tuple = (1.4, 5.3)
print("Float tuple:", my_float_tuple)

# Storing a tuple of strings (multiple strings seperated by commas)
my_string_tuple = ("dude", "totally", "yeah")
print("String tuple:", my_string_tuple)

Output:
Integer tuple: (3, 5, 8, 2)
Float tuple: (1.4, 5.3)
String tuple: ("dude", "totally", "yeah")
