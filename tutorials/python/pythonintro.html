Intro to Python

A little about Python

Python is an easy to use programming language. It's designed to resemble natural spoken language rather than the usual, more machine-oriented, programming languages like C++ and Java. That makes it easier to read and understand, and also makes it very easy to learn.

While Python is pretty simple, it happens to also be incredibly powerful and makes the process of getting your ideas up and running quicker than most others.

Some notable features of Python:

  • It's elegant, easy to read and write.
  • It has a large amount of pre-written, open-source packages. So you don't have to "reinvent the wheel".
  • It's expandable and open-source, allowing you to alter the very language itself to suit your needs.
  • It's free!

It's free

Not only is Python free to use, it's also free in the sense that it's open-source. Open-source means that you, or any anyone, are welcome to change the structure and functionality of it's source code . While it's not necessary to do that, it can be very useful.

It's popular

Python is actually an unbelievably popular language all around the world. Many jobs are available to those who gain an understanding of it and really get to know how to use it. Actually it happens to be the #1 most popular programming language in the world, and that's probably not going to change any time soon.

It's object-oriented

If you aren't familiar with other programming languages you may not have heard of the phrase "object-oriented". However it's a simple concept which most programming languages are designed around, and it's pretty self explanatory, but here's the scoop.

The idea that object-oriented programming languages revolve around is, well, objects! Just like real world objects these virtual objects have qualities which define them. Say you're making a text based game, where you have a player 'object'. This player object would have some data that defines it. Like a name perhaps, and if it's an adventure game maybe the player should have some stats. Let's say stamina, and health. An example for the layout of this player object could look like this:

Player:
  • has a name
  • has an amount of health
  • has a stamina stat

This is a very basic example, and objects can be used in many ways. This player object would likely be an object that only gets instantiated, or 'spawned', once. Some objects may instead be used as a template for many objects in your game.For this text adventure themed example, you might have an object defining what attributes each item you put into your game would have. This item object may have a name and an id to represent what kind of item it is, and the layout for that might look something like this:

Item:
  • has a name
  • has a type id

This Item object could then be created and used over and over as a template for every item you put into your game. Each of those items would then be able to have their own name and type that can be retrieved for use later in your game. Like when listing the items that the player has in their inventory or any other case you find to need it. A couple examples might look like this,

Item: Steel Sword
  • has a name: "Steel Sword"
  • has a type id: "Weapon"
Item: Health Potion
  • has a name: "Health Potion"
  • has a type id: "Potion"

Now again this is a simplified way of explaining it, but I think it gets the idea across without getting too technical, it's necessary to mention that these objects can, and should, also have some 'ways' to do things. With that said we might then extend the Player object layout above to something more like:

Player:
  • has a name
  • has an amount of health
  • has a stamina stat
  • has a 'way' to change it's name
  • has a 'way' to adjust it's stamina stat
  • has a 'way' to adjust it's health

Of course these layouts above aren't written in actual code, but instead what we call pseudocode. Pseudocode is just a fancy word for describing the steps that code takes to accomplish a task. It can make it easier to plan and share what your code is doing in a more comprehensive way than writing it all out in code just to get an idea across. I've put 'way' in quotations to express that it stands for a Function in Python but we will go over those later in detail.

One of these object templates in Python would be referred to as a class. So when we talk about classes, we are referring to object templates essentially. Here we really won't go much more into detail about these things as this is only meant to be an introduction.
